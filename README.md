# TP 23 – Migration de Eureka vers Consul
# Par: Rim EL ABBASSI

Ce travail pratique présente la migration d’un mécanisme de **service discovery** basé sur **Eureka** vers **Consul**, dans une architecture microservices Spring Boot.  
L’objectif est de comprendre le rôle de Consul comme **source de vérité** pour l’enregistrement, la découverte et l’état de santé des services.

---

## Objectifs

- Comprendre la logique d’une migration de service discovery (**Eureka → Consul**)
- Configurer **Consul** pour l’enregistrement et la découverte des microservices
- Supprimer la dépendance à Eureka dans les services
- Déployer l’ensemble avec **Docker** et **Docker Compose**
